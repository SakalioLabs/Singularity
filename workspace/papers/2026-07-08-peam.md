# Paper Card: PEAM

**Title**: PEAM: Parametric Embodied Agent Memory through Contrastive Internalization of Experience in Minecraft
**Source**: https://arxiv.org/abs/2605.27762
**Date**: 2026
**Authors**: Yuchen Guo, Junli Gong, Hongmin Cai, Yiu-ming Cheung, Weifeng Su
**Type**: Paper
**Core Problem**: Retrieval memory helps embodied agents, but inference-time retrieval can become slow, brittle, and disconnected from reflexive execution.
**Core Method**: Parametric embodied memory that consolidates selected experiences into fast model parameters while keeping a slower deliberative LLM for reasoning.
**System Architecture**:
  - Slow deliberative LLM handles open-ended reasoning.
  - Episodic staging memory collects success and failure-correction traces.
  - Fast MoE-LoRA parametric module internalizes selected skills.
  - Parameterization-worthiness and self-triggered consolidation decide what and when to learn.
**Memory Mechanism**: Hybrid explicit episodic memory plus parameter-resident skill memory.
**Task Mechanism**: Consolidate repeated or valuable trajectories into a fast execution layer.
**Key Results**: Reports better long-horizon Minecraft performance, better retention of consolidated skills, and lower reliance on retrieval at inference time.
**Reproduction Difficulty**: High; requires training LoRA/MoE modules and trajectory datasets.
**Borrowable Points**:
  - Treat failure-correction pairs as first-class learning data.
  - Add a consolidation-worthiness score before promoting traces.
  - Keep a slow planner plus fast actor split.
**Current Singularity Mapping**:
  - `ExperienceRecord` stores success/failure signals and causal fields.
  - `SkillExtractor` already computes a consolidation score.
  - `RuntimeSupervisor` is the first slow-planner/fast-actor split.
**Next Action**: Extend skill-candidate scoring with repeated-failure correction bonuses and latency savings.
