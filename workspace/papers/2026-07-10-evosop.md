# Paper Card: EvoSOP

**Title**: From Atomic Actions to Standard Operating Procedures: Iterative Tool Optimization for Self-Evolving LLM Agents

**Source**: https://arxiv.org/abs/2607.07321

**Date Reviewed**: 2026-07-10

**Core Method**: Synthesizes recurring atomic-action trajectories into callable higher-order SOP tools and manages them through construction, merging, evaluation, and pruning.

**Evidence**:
- Reports improved task success and fewer interaction rounds than static atomic-tool baselines.
- Treats tool optimization as an iterative lifecycle rather than one-shot trajectory extraction.

**Useful Transfer**:
- Minecraft composite skills should represent verified state transitions, not merely remembered action strings.
- Merge only equivalent precondition/postcondition contracts.
- Promotion, refinement, and pruning need separate evidence and rollback paths.

**Project Action**: Frontier routing can now select prerequisite-closing composite skills by transition fit. SOP synthesis and merging remain review-gated; future work should use repeated verified routes and the existing skill lifecycle rather than automatically converting every successful trace.
