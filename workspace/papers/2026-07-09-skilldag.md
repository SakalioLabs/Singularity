# Paper Card: SkillDAG

**Title**: SkillDAG: Self-Evolving Typed Skill Graphs for LLM Skill Selection at Scale
**Source**: https://arxiv.org/abs/2606.03056
**Date**: 2026
**Type**: Skill retrieval / skill graph framework
**Core Problem**: Large skill libraries become hard to search with flat lists or embedding similarity because skills depend on, specialize, duplicate, or conflict with each other.
**Core Method**: Models inter-skill relationships as a typed directed graph exposed to the agent as an inference-time retrieval interface, then lets execution-backed edges accumulate over episodes.
**Memory / Task Mechanism**:
  - Treats skills as graph nodes with typed edges rather than independent snippets.
  - Combines vector matches, graph neighbors, and conflict signals during retrieval.
  - Uses propose-then-commit updates so the graph evolves from execution evidence.
**Borrowable Points**:
  - Add a typed skill graph audit before scaling beyond a small custom skill set.
  - Track dependencies, missing prerequisites, postconditions, and governance status per skill.
  - Keep edge creation evidence-backed, especially for experiment-derived skills.
**Singularity Adaptation**:
  - `Skill` now has structured `dependencies`, `provenance`, and `gate` fields for custom reviewed skills.
  - `skill-graph-report` audits built-in and custom skills for dependencies, missing edges, postconditions, cycles, and gate/provenance coverage.
  - Future retrieval can use typed graph neighbors after live logs provide enough execution-backed edges.
**Next Action**: Run `skill-graph-report` after real skill-candidate approvals and use missing/ungoverned nodes to prioritize review.
