# Paper Card: Solaris

**Title**: Solaris: Building a Multiplayer Video World Model in Minecraft
**Source**: https://arxiv.org/abs/2602.22208
**Date**: 2026
**Type**: Multiplayer Minecraft data engine / video world model
**Core Problem**: Existing Minecraft agent frameworks either provide controllable high-level actions without rich visuals or visual observations without scalable controllable multiplayer behavior.
**Core Method**: SolarisEngine combines Minecraft server orchestration, camera/controller bots, Mineflayer-style primitives, and a multiplayer communication layer to collect cooperative gameplay episodes with visual observations.
**Memory / Task Mechanism**:
  - Uses reusable skill primitives to script cooperative episodes.
  - Captures multiplayer state/action/video traces for future world-model and VLA training.
  - Highlights the gap between high-level API control and visually grounded multiplayer evaluation.
**Borrowable Points**:
  - Treat multi-bot visual trace collection as a first-class benchmark artifact, not a side effect of live runs.
  - Keep role-specific bridge ports, usernames, and shared-state logs explicit so later world-model training can align actions to observations.
  - Audit whether successful controller actions actually change world state before trusting them as progress.
**Singularity Adaptation**:
  - M7 role bridge plans, screenshot bridge support, visual trace reports, and collaboration schedule comparisons already cover much of the orchestration layer.
  - `self-evolution-report` now discounts successful actions that do not produce observed state, inventory, or verifier deltas, preventing API success returns from inflating progress.
  - Future multiplayer screenshot traces should join role action logs, shared-state revisions, and verified image frames into a Solaris-style dataset profile before training any world-model or VLA fallback.
**Next Action**: Run M7 with screenshot-capable role bridges, then compare action success, state deltas, and visual frames before treating the run as training-quality multiplayer data.
