# Paper Card: MemGym

**Title**: MemGym: a Long-Horizon Memory Environment for LLM Agents

**Source**: https://arxiv.org/abs/2605.20833

**Date Reviewed**: 2026-07-10

**Core Method**: Unifies long-horizon agent environments behind a memory-reasoning interface and reports memory-isolated scores intended to separate memory quality from reasoning, retrieval, and tool-use capability.

**Evidence**:
- Covers tool-use dialogue, deep research, coding, and computer-use regimes.
- Uses length-controllable, stage-ablated synthetic pipelines for its CODEQA and deep-research tracks.
- Trains a lightweight reward model to estimate compression quality where full environment rollouts are expensive.

**Useful Transfer**:
- Compare task-continuity lineage enabled versus disabled while holding the Minecraft planner, action backend, verifier, seeds, and task stream fixed.
- Report memory-local outcomes such as active-path precision, failed-branch contamination, context characters, valid checkpoint recovery, and downstream verifier deltas.
- Keep compression quality separate from task success so concise but incorrect checkpoints cannot earn promotion.

**Do Not Copy Blindly**:
- MemGym does not establish Minecraft world-state restoration safety.
- Synthetic memory tasks and learned reward scores cannot replace in-world reachability or inventory/state verification.
- A memory-only gain still needs repeated held-out Minecraft transfer evidence before runtime promotion.

**Project Action**: `task-continuity-lineage-ablation` now freezes planner, action backend, verifier, task stream, and seed identifiers while comparing flat versus lineage-selected memory. Its built-in cases improve active-path precision and remove failed-branch contamination but use more context, so they support correctness only, not efficiency. A separate shadow-restoration report and default-live gate require repeated non-regressing evidence before shadow selection; automatic restoration remains impossible.
