# Paper Card: GovMem

**Title**: When Not to Write Memory: Governing False Promotion from Correlated Agent Traces
**Source**: https://arxiv.org/abs/2607.02579
**Date**: 2026
**Type**: Paper
**Core Problem**: Multi-agent and long-running agents can falsely promote repeated claims into durable memory even when the repeated evidence comes from copied sources, shared prompts, shared tools, stale context, or a narrow scope.
**Core Method**:
  - Reframes memory writing as evidence governance, not just retrieval optimization.
  - Separates promote, reject, and needs-review routes for candidate memories.
  - Treats dependency-aware support, counterevidence, temporal validity, and scope as write-path requirements.
**Borrowable Points**:
  - Repeated observations are not automatically independent evidence.
  - Memory candidates should carry provenance/dependency fields before they can become durable facts.
  - Conservative review gates are preferable to silently adding stale or correlated facts to shared memory.
**Singularity Adaptation**:
  - `MemoryLifecyclePolicy` now flags candidate writes with `correlated_evidence` when metadata reports copied/shared/low-trust evidence dependencies.
  - It flags `unsafe_scope` when candidate validity is stale, out of scope, adversarial, or contradicted.
  - Advisory mode preserves writes but marks them review-needed; `enforce_memory_write_gate` can suppress those candidates after enough trace evidence exists and an approved memory-promptware gate is supplied.
  - Future multi-agent traces should populate dependency, evidence status, and scope metadata so M7 shared memory avoids false promotion.
**Next Action**: Add provenance/dependency fields to multi-agent memory candidates and compare false-promotion review counts on M7 collaboration logs.
