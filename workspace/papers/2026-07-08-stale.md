# Paper Card: STALE

**Title**: STALE: Can LLM Agents Know When Their Memories Are No Longer Valid?
**Source**: https://arxiv.org/abs/2605.06527
**Date**: 2026
**Type**: Paper + benchmark
**Core Problem**: Long-term memory agents often retrieve updated evidence but still act on outdated assumptions when new observations implicitly invalidate older memories.
**Core Method**:
  - Introduces implicit conflict as a memory failure mode.
  - Evaluates state resolution, premise resistance, and implicit policy adaptation.
  - Proposes state-aware consolidation and propagation-aware search as an initial direction.
**Borrowable Points**:
  - Memory should track mutable state, not just facts.
  - Later observations can supersede prior memories without explicit contradiction text.
  - Agents need provenance plus supersession history so stale assumptions can be rejected at write time and read time.
**Singularity Adaptation**:
  - `MemoryLifecyclePolicy` now flags `state_revision` and `implicit_conflict` candidates for review.
  - M7 shared-state execution compares new shared-memory writes against prior per-key provenance history and records `supersedes` metadata when a value changes.
  - Collaboration reports now count state revisions and implicit conflicts alongside GovMem-style false-promotion review counts.
**Next Action**: Run live M7 and autonomous traces with mutable world-state keys, then compare how often stale assumptions are detected before they affect task selection.
