# Paper Card: AgentCL

**Title**: AgentCL: Toward Rigorous Evaluation of Continual Learning in Language Agents
**Source**: https://arxiv.org/abs/2606.02461
**Date**: 2026
**Type**: Continual-learning evaluation framework for language agents
**Core Problem**: Agents often spend large inference budgets solving one task but underuse experience in later tasks, and naive task streams make memory-design comparisons hard to interpret.
**Core Method**: Builds controlled compositional task streams where earlier sub-solutions, evidence, and workflows are intentionally reusable later, then measures transfer gains and memory-induced degradation.
**Memory / Task Mechanism**:
  - Evaluates whether stored interactions, insights, and skills transfer to later tasks.
  - Filters unreliable experiences during consolidation.
  - Contrasts controlled reusable streams with naive streams.
**Borrowable Points**:
  - Minecraft evaluation should distinguish generic progress from reusable cross-task transfer.
  - Memory and skill candidates should be judged against later tasks where their prerequisites or sub-solutions reappear.
  - Reports should include both plasticity and interference/degradation signals.
**Singularity Adaptation**:
  - `continual-learning-report` now surfaces memory read/write loops, object/action exploration, and meaningful-horizon scores as the aggregate diagnostic layer.
  - Next, controlled Minecraft task streams should compare whether transfer memories and approved skills improve held-out crafting, mining, building, and navigation variants.
**Next Action**: Add controlled task-stream cases that pair reusable sub-solutions with held-out Minecraft goals and measure transfer gains versus memory-induced regressions.
