# Paper Card: Voyager

**Title**: Voyager: An Open-Ended Embodied Agent with Large Language Models
**Source**: https://arxiv.org/abs/2305.16291
**Date**: 2023
**Authors**: Guanzhi Wang, Yuqi Xie, Yunfan Jiang, Ajay Mandlekar, Chaowei Xiao, Yuke Zhu, Linxi Fan, Anima Anandkumar (NVIDIA + Caltech)
**Type**: Paper + Code
**Core Problem**: How to build an open-ended, continually-learning agent in Minecraft?
**Core Method**: LLM-driven automatic curriculum + code-as-skill library + self-verification
**System Architecture**:
  - Automatic Curriculum: GPT-4 proposes progressively harder tasks based on agent state
  - Skill Library: Successful code solutions stored as reusable JavaScript functions
  - Iterative Prompting: GPT-4 generates code, executes, receives feedback, revises
  - Self-Verification: GPT-4 checks if task was completed by examining environment state
**Input**: Environment state (block/entity info), skill library, current task
**Output**: JavaScript code (Mineflayer API calls)
**Action Space**: JavaScript code generation (Mineflayer API)
**Memory Mechanism**: Code-based skill library (permanent), environment feedback (ephemeral)
**Task Mechanism**: Automatic curriculum (no fixed task set)
**Skill Mechanism**: Code generation + library storage + retrieval by similarity
**Evaluation Tasks**: Item unlocking, tech tree traversal, exploration distance
**Key Results**:
  - Unlocked 63 unique items (vs 33 for ReAct, 38 for Reflexion)
  - Traveled 2.3x longer distances than baselines
  - Retrieved relevant skills 95.7% of the time
  - Self-verification reduced hallucination artifacts
**Failure Cases**:
  - Generated code sometimes has subtle bugs (infinite loops, wrong block targeting)
  - Curriculum can get stuck on hard tasks
  - Requires expensive GPT-4 API calls
  - Sandboxed but still risky code execution
**Dependencies**: Mineflayer, MineDojo environment, OpenAI GPT-4 API
**Is Open Source**: Yes (Apache 2.0) - https://github.com/MineDojo/Voyager
**Reproduction Difficulty**: Medium (MineDojo dependency, specific environment version)
**Borrowable Points**:
  - Code-as-skill library design (highly reusable)
  - Self-verification pattern
  - Automatic curriculum concept
**Cannot Directly Adopt Because**:
  - MineDojo environment is heavy and complex
  - GPT-4 dependency (expensive)
  - Archived project (not maintained)
**Suggestion for This Project**:
  - Adopt skill library design directly
  - Implement self-verification without MineDojo
  - Use Mineflayer directly instead of MineDojo wrapper
  - Start with manual curriculum, auto-curriculum later
**Next Action**: Implement basic skill library inspired by Voyager's design
