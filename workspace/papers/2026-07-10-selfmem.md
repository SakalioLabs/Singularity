# Paper Card: SelfMem

**Title**: SelfMem: Self-Optimizing Memory for AI Agents

**Source**: https://arxiv.org/abs/2607.03726

**Date Reviewed**: 2026-07-10

**Core Method**: Exposes memory tools and feedback signals so an agent can explore, evaluate, and refine its memory strategy instead of relying on one fixed storage, retrieval, and summarization policy.

**Evidence**:
- Evaluated on BEAM from 100K to 1M-token conversation scales.
- Reports official-score gains of 48.7%, 40.8%, and 41.9% over the strongest baseline at 100K, 500K, and 1M tokens.
- Model-guided strategy refinement improves results beyond a fixed initial strategy.

**Useful Transfer**:
- Treat memory policy as an evaluated candidate, not an immutable hand-tuned constant.
- Feed retrieval outcome, context cost, freshness, and downstream task success back into offline strategy selection.
- Preserve task-specific memory strategies rather than forcing one retrieval layout onto every Minecraft task family.

**Do Not Copy Blindly**:
- Conversation-scale memory scores do not prove embodied action value.
- Runtime self-modification must remain review-only until controlled Minecraft traces and held-out transfer gates pass.
- Strategy optimization must not bypass promptware filtering, provenance, or capability evidence gates.

**Project Action**: Use the new per-decision planning-memory contract as the observable substrate for future memory-policy variants. It bounds typed memory reads, not the whole planner prompt. Compare variants offline, then promote only gate-approved budgets and typed-layer mixes.
