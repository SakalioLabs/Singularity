# Paper Card: ActMem

**Title**: ActMem: Bridging the Gap Between Memory Retrieval and Reasoning in LLM Agents
**Source**: https://arxiv.org/abs/2603.00026
**Date**: 2026
**Type**: Paper + evaluation dataset
**Core Problem**: Agent memory is often treated as passive retrieval, so agents may recall facts without reasoning through their causal implications, constraints, or conflicts.
**Core Method**:
  - Transforms histories into a causal and semantic memory graph.
  - Uses counterfactual reasoning and commonsense completion for conflict resolution.
  - Evaluates logic-driven scenarios where memory must affect decisions, not just answers.
**Borrowable Points**:
  - Retrieved memories should be action-conditioned and conflict-aware.
  - Causal graph structure helps detect implicit constraints that raw text retrieval misses.
  - Memory evaluation should inspect whether retrieved evidence changes downstream decisions.
**Singularity Adaptation**:
  - Existing WISE-style `CausalEventIndex` is the right substrate for ActMem-style actionable retrieval.
  - Read-time memory filtering now prevents stale durable entries from entering planner prompts before causal reasoning.
  - Future work should rank durable entries, transferable experiences, and causal events together as action-conditioned evidence.
**Next Action**: Combine `memory_read_filter_report` with causal event retrieval to measure whether filtered memory changes task/action selection.
