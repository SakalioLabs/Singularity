# Paper Card: OpenHA / CrossAgent

**Title**: OpenHA: A Series of Open-Source Hierarchical Agentic Models in Minecraft
**Source**: https://arxiv.org/abs/2509.13347
**Related**: https://arxiv.org/abs/2512.09706
**Code**: https://github.com/CraftJarvis/OpenHA
**Date**: 2025-2026
**Type**: Paper + code + model/dataset release
**Core Problem**: Minecraft agents need action spaces that support both high-level reasoning and low-level precise control; no single abstraction is universally optimal across tasks.
**Core Method**:
  - Compares multiple abstracted action spaces and tokenizers for hierarchical/VLA Minecraft agents.
  - Introduces Chain of Action, where abstracted actions act as intermediate reasoning steps that guide final executable actions.
  - CrossAgent extends the direction with heterogeneous action spaces and dynamic interface switching.
**Borrowable Points**:
  - Treat canonical actions as inspectable intermediate reasoning, not merely backend commands.
  - Keep multiple backends viable: Mineflayer API for reliability, desktop/keyboard-mouse for visual generalization, and abstract action traces for learning.
  - Benchmark when a task benefits from high-level API control versus low-level visual control.
**Singularity Adaptation**:
  - `ActionMapper` already separates canonical actions from backend commands.
  - `action-abstraction-report` now reads session logs and compares canonical action usage, observed Mineflayer/backend commands, planned desktop mappings, failed mappings, and lower-level visual-control candidates.
  - `ActionGranularityPolicy` consumes those hints and records `control_policy` decisions while keeping Mineflayer as the safe fallback until desktop control is executable.
**Next Action**: Run the report on real benchmark traces and compare policy-guided fallback decisions against action failures by task family.
