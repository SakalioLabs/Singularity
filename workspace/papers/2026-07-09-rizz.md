# Paper Card: RIZZ

**Title**: RIZZ: Routing Interactions to Near Zero-Interference Zones for Continual Adaptation of Black-Box Agents
**Source**: https://arxiv.org/abs/2606.20638
**Date**: 2026
**Type**: Verifier-gated routed memory for continual black-box adaptation
**Core Problem**: Long-lived agents can improve from memory, but undifferentiated global memory can contaminate unrelated task families and cause interference.
**Core Method**: Routes interactions into dynamically managed memory zones, retrieves branch-local/global/working context under a budget, and updates memory only after verifier feedback.
**Memory / Task Mechanism**:
  - Keeps branch-local examples, failures, and procedural rules.
  - Uses verifier scores to promote reusable rules, demote harmful rules, or create anti-patterns.
  - Controls interference by routing similar tasks together while preventing unrelated memory spillover.
**Borrowable Points**:
  - Minecraft task families such as mining, crafting, shelter, navigation, and redstone should be evaluated for cross-family interference before sharing memories.
  - Promotion gates need explicit reject paths for harmful transfer, not only approval/review.
  - Anti-pattern memories should be recorded when a reusable rule hurts held-out or second-pass tasks.
**Singularity Adaptation**:
  - `task-stream-transfer-gate` rejects promotion when controlled streams show interference, stability regression, or held-out regression.
  - `MemoryLifecyclePolicy` now blocks semantic/experience promotion when a transfer gate is rejected or errored, and routes review gates to operator review.
  - Existing stale/conflict memory filters can become branch-local once real task-stream data identifies families that interfere.
**Next Action**: Route transfer-memory and skill retrieval through task-family zones after real stream specs expose cross-family interference.
