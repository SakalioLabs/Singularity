# Paper Card: ReAct

**Title**: ReAct: Synergizing Reasoning and Acting in Language Models
**Source**: https://arxiv.org/abs/2210.03629
**Date**: 2022
**Authors**: Shunyu Yao, Jeffrey Zhao, Dian Yu, Nan Du, Izhak Shafran, Karthik Narasimhan, Yuan Cao (Princeton + Google)
**Type**: Paper
**Core Problem**: LLMs that only reason or only act have limitations. Can interleaving both help?
**Core Method**: Interleaved reasoning traces and action execution. Agent thinks, acts, observes, thinks again.
**Key Results**:
  - Significant improvement on knowledge-grounded reasoning tasks
  - More interpretable and grounded than pure reasoning
  - Actions ground reasoning in real environment feedback
  - Works across HotpotQA, Fever, ALFWorld, WebShop
**Borrowable Points**:
  - Thought-Action-Observation loop is exactly our agent architecture
  - Reasoning traces improve action quality
  - Observation feedback corrects reasoning errors
  - Pattern is domain-agnostic
**Cannot Directly Adopt**:
  - ReAct uses text actions, not structured game actions
  - No long-term memory or skill system
**Suggestion**: Our agent already follows ReAct pattern. Use explicit reasoning traces in planning prompts.
**Next Action**: Add explicit "thinking" step in our planner prompts
