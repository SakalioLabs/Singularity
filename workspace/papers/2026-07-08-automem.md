# Paper Card: AutoMem

**Title**: AutoMem: Automated Learning of Memory as a Cognitive Skill
**Source**: https://arxiv.org/abs/2607.01224
**Date**: 2026
**Type**: Paper
**Core Problem**: Long-horizon agents need to decide what to remember, when to retrieve it, and how to organize external memory without relying on manual review of entire trajectories.
**Core Method**:
  - Treats memory management as a cognitive skill rather than a fixed retrieval module.
  - Promotes file-system memory operations to first-class actions alongside task actions.
  - Uses two loops: scaffold revision for memory structure and targeted training from good memory decisions found in prior episodes.
**Borrowable Points**:
  - Make memory operations explicit, logged, and evaluable.
  - Review complete trajectories offline to discover better memory schemas and retrieval habits.
  - Separate memory scaffold quality from task-action policy quality so improvements can be measured independently.
**Singularity Adaptation**:
  - Current memory reports already track recall diversity and consolidation candidates.
  - `memory-policy-report` now classifies explicit memory writes/reads/manage events plus inferred context, episodic, semantic, failure-learning, noisy-write, and consolidation signals from session logs.
  - `memory_policy_feedback` mirrors the `action_abstraction_feedback` pattern and can be applied to future memory-write/retrieval policy objects.
  - Agent runtime now emits structured `memory_write`, `memory_read`, and `memory_manage` events for the report.
**Next Action**: Run `memory-policy-report` on real benchmark logs and tune semantic write gates against noisy-write and missed-write feedback.
