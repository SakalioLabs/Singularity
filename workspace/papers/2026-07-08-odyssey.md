# Paper Card: Odyssey

**Title**: Odyssey: Empowering Minecraft Agents with Open-World Skills
**Source**: https://arxiv.org/abs/2407.15325
**Venue**: IJCAI 2025
**Authors**: Shunyu Liu, Yaoru Li, Kongcheng Zhang, Zhenyu Cui, Wenkai Fang, Yuxuan Zheng, Tongya Zheng, Mingli Song
**Type**: Paper + code/dataset
**Core Problem**: Minecraft agents often overfit to narrow tech-tree tasks such as ObtainDiamond and lack diverse open-world skills.
**Core Method**: Planner-actor-critic architecture with an open-world skill library, a Minecraft-Wiki-derived QA model, and capability benchmarks for long-term planning, dynamic-immediate planning, and autonomous exploration.
**Memory / Skill Mechanism**:
  - 40 primitive skills and 183 compositional skills.
  - Skill retrieval maps planner subgoals to executable skill code.
  - Critic evaluates execution through self-validation and reflection using environment feedback and inventory changes.
**Borrowable Points**:
  - Treat subgoal verification as a critic function, not as part of action execution.
  - Use inventory deltas and environment feedback as low-cost validation signals before invoking a heavy LLM/VLM critic.
  - Recursive skill prerequisites can reduce planner burden for common tech-tree procedures.
**Singularity Adaptation**:
  - `GoalVerifier` is the deterministic first critic: it checks inventory, safety, shelter, food/health, and exploration evidence before accepting common completions.
  - Recipe and resource-drop anchors are now generated from structured Minecraft knowledge, reducing the amount of hand-written critic logic.
  - Skill-library postconditions now generate anchors too, and post-action before/after inventory deltas are preserved as validation evidence.
  - Verifier results now reject weak skill candidates before promotion into the reviewed skill library.
  - Skill-level self-validation reports now explain why a candidate was approved, rejected, or left unknown.
  - Benchmark ingestion now aggregates those self-validation reports across a suite, exposing approved/rejected/unknown readiness and verifier status counts before review.
  - Unknown verifier outcomes can now call an explicit LLM critic before promotion, mirroring Odyssey's critic role while keeping deterministic verifier evidence as the first gate.
  - Unknown runtime goal completions can now call an opt-in `GoalVerificationCritic`, so Odyssey-style critic judgment covers both skill promotion and live planner completion.
  - `goal-verification-ablation` now replays session-log goals through deterministic-only, API visual summary, and screenshot/VLM-assisted critic paths before changing online completion policy.
**Next Action**: Run live M1/M2 suites with deterministic-only versus critic-assisted goal verification and compare false-complete rejection rates.
