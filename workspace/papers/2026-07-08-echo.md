# Paper Card: Echo

**Title**: Experience Transfer for Multimodal LLM Agents in Minecraft Game
**Source**: https://arxiv.org/abs/2604.05533
**Secondary Source**: https://openaccess.thecvf.com/content/CVPR2026/papers/Li_Experience_Transfer_for_Multimodal_LLM_Agents_in_Minecraft_Game_CVPR_2026_paper.pdf
**Date**: 2026
**Authors**: Chenghao Li, Jun Liu, Songbo Zhang, Huadong Jian, Hao Ni, Lik-Hang Lee, Sung-Ho Bae, Guoqing Wang, Yang Yang, Chaoning Zhang
**Type**: Paper
**Core Problem**: Minecraft agents need to reuse prior experience across related tasks instead of storing static episode logs.
**Core Method**: Transfer-oriented memory with five reusable dimensions: structure, attribute, process, function, and interaction.
**System Architecture**:
  - Contextual state descriptors structure task experience.
  - In-context analogy learning retrieves and adapts similar experiences.
  - Memory is treated as reusable procedure and environment knowledge.
**Memory Mechanism**: Experience atoms decomposed into transfer dimensions.
**Task Mechanism**: Use analogy to unlock related objects faster after learning one task.
**Key Results**: Reports speed-ups on Minecraft object-unlocking tasks and chain-like transfer to related tasks.
**Reproduction Difficulty**: Medium-high; multimodal setup and benchmark details matter.
**Borrowable Points**:
  - Store structure/process/function/interaction explicitly.
  - Retrieve experiences by transferable dimensions, not only keyword overlap.
  - Track when a learned task unlocks a family of related tasks.
**Current Singularity Mapping**:
  - `ExperienceRecord.dimensions` follows this structure.
  - `SkillExtractor.extract_experience_atoms()` derives experience atoms from session logs.
  - `MemorySystem.rank_transfer_experiences()` now scores structure, attribute, process, function, and interaction matches separately and exposes `transfer-memory-report` for offline retrieval audits.
  - `skill-memory-quality-report` now checks whether transferred skill-memory hints are followed by success, repeated failures, or review-only outcomes before they are trusted as default reusable experience.
**Next Action**: Run `transfer-memory-report` and `skill-memory-quality-report` on real M1/M2/autonomous memory stores, then compare transfer-axis matches against task outcomes.
