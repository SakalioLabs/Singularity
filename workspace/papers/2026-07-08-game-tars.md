# Paper Card: Game-TARS

**Title**: Game-TARS: Pretrained Foundation Models for Scalable Generalist Multimodal Game Agents
**Source**: https://arxiv.org/abs/2510.23691
**Project**: https://seed-tars.com/game-tars/
**Date**: 2025
**Authors**: Zihao Wang et al.
**Type**: Paper + Foundation Model
**Core Problem**: Game agents trained on narrow APIs or single environments do not generalize well across heterogeneous visual games.
**Core Method**: Generalist multimodal game agent pre-trained with unified keyboard-mouse action space and large-scale cross-game data.
**System Architecture**:
  - Human-aligned native keyboard-mouse action representation.
  - Large-scale multimodal trajectory pretraining.
  - Decaying continual loss reduces causal confusion.
  - Sparse-thinking balances reasoning depth and inference cost.
**Memory Mechanism**: Primarily parametric pretraining; not an explicit episodic-memory agent.
**Task Mechanism**: Scalable action representation transfers across game genres and interaction modes.
**Key Results**: Reports strong open-world Minecraft and cross-game generalization.
**Reproduction Difficulty**: Very high; foundation-model scale.
**Borrowable Points**:
  - Keep action representation portable beyond Mineflayer APIs.
  - Use sparse reasoning: reason deeply only when the state demands it.
  - Track causal confusion between action and outcome in training/evaluation logs.
**Current Singularity Mapping**:
  - Mineflayer API path is practical for M1-M5.
  - Future desktop-control path should map actions to a unified keyboard-mouse schema.
**Next Action**: Add an action abstraction layer that can later target either Mineflayer API or desktop inputs.
