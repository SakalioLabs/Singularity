# Paper Card: MemTier

**Title**: MemTier: Tiered Memory Architecture and the Retrieval Bottleneck in Long-Running LLM Agents
**Source**: https://arxiv.org/abs/2605.03675
**Date**: 2026
**Type**: OpenClaw runtime memory architecture
**Core Problem**: Long-running agents accumulate too much memory, making retrieval noisy, stale, and weakly tied to later task outcomes.
**Core Method**:
  - Uses a tiered architecture with structured episodic storage, weighted retrieval, attribution-based weight updates, and asynchronous consolidation.
  - Treats retrieval quality as a runtime learning problem rather than a fixed BM25/vector query.
  - Links tool outcomes back to memory weighting so useful memories become easier to retrieve.
**Borrowable Points**:
  - Keep retrieval changes attributable to explicit outcome evidence.
  - Separate active runtime retrieval from slower consolidation/promotion.
  - Diagnose memory weighting before mutating durable semantic memory.
**Singularity Adaptation**:
  - `memory-attribution-report` now labels generic planner `memory_read` events as supported, conflicting, review, or no-result using downstream plan/action/goal outcomes without printing raw queries.
  - `memory-attribution-gate` blocks weighted memory retrieval profiles until saved attribution reports meet supported-read, conflict-rate, and no-result-rate thresholds.
  - `skill-memory-quality-report` attributes typed skill-memory hints to later action/goal outcomes.
  - `skill-memory-quality-gate` now decides whether localized `REUSE` evidence is approved, review-only, or rejected before feedback can become a runtime profile.
  - Runtime `--skill-memory-quality-gate` loading follows MemTier's outcome-attributed retrieval update idea while preserving Singularity's conservative non-mutating skill store.
**Next Action**: Run both attribution gates on real M1/M2/autonomous logs, then compare gated retrieval-quality runs against baseline memory context and ungated feedback ablations.
