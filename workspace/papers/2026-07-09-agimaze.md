# Paper Card: AGI Maze

**Title**: AGI Maze as a Benchmark Framework for World-Modeling Agents
**Source**: https://arxiv.org/abs/2607.00627
**Date**: 2026
**Type**: World-modeling benchmark / partially observable grid environments
**Core Problem**: LLM agents can appear to reason in text while failing to maintain a persistent, manipulable representation of partially observable external state.
**Core Method**: Builds lightweight maze environments with multiple difficulty regimes where agents must learn and use world-state representations rather than infer only from local observations.
**Memory / Task Mechanism**:
  - Separates local observations from the latent map the agent must construct over time.
  - Treats message history as a weak working-memory baseline, not a sufficient world model.
  - Emphasizes step budgets and hidden-state representation as evaluation pressure.
**Borrowable Points**:
  - Minecraft exploration traces should produce an explicit world-state artifact, not only a narrative memory summary.
  - Visited cells, transitions, unseen frontiers, resource hotspots, and danger cells should be auditable from logs.
  - Curriculum goals can be generated from map gaps and frontier cells rather than only novelty text.
**Singularity Adaptation**:
  - `world-model-report` now reconstructs X/Z cells, transitions, resource hotspots, danger cells, frontier candidates, and suggested exploration goals from session logs.
  - The report emits `world_model_feedback`, and `CurriculumManager.record_world_model_feedback()` converts map gaps, resource hotspots, and danger cells into ranked autonomous curriculum candidates.
  - `world-model-feedback-gate` now requires structured frontier/resource/danger evidence before saved map feedback can enter runtime curriculum scoring through `--world-model-feedback`.
  - This complements MineExplorer-style coverage metrics by checking whether the agent has an actionable world-state map for the next exploration step.
**Next Action**: Run `world-model-report` plus `world-model-feedback-gate` on real autonomous/M7 session logs, then compare approved frontier suggestions against subsequent exploration success and stuck-path failures.
